# cocopy

Chrome extension (Manifest V3) that copies text from a web page through user-defined JavaScript functions.

## Tech Stack

TypeScript / React 18 / styled-components / Webpack 5 / Jest / gts (lint).

See `package.json` for scripts and `README.md` for end-user usage.

## Architecture

The defining design constraint: **user-defined code must never run in the extension context.** All evaluation happens inside `src/sandbox.ts`, an isolated iframe that communicates with popup/options via `postMessage`. The protocol and result type (`CopyResult`) live in `src/lib/eval.ts`; the function shape lives in `src/lib/function.ts`.

Storage uses `chrome.storage.sync` (`src/lib/config.ts`). Functions are validated against `src/lib/function.schema.json`; the compiled validator `src/lib/function.ajv.js` is **checked in** — regenerate with `yarn validator` after schema edits or tests will drift.

`gallery/` is a sub-project. `gallery/gallery.yaml` is the source of truth; `gallery/README.md` is generated by `yarn update-gallery` and should not be hand-edited.

## Verifying Changes

- `yarn test` — Jest tests, then `gts check` via `posttest`. A lint failure fails the command; `yarn fix` auto-fixes most issues.
- `yarn build` — confirms the extension compiles.
